Test de compréhension#

Question 1

Apache Airflow sert à :

  • Créer des dashboards
  • Orchestrer des workflows
  • Compiler du code
  • Installer un OS

Question 2

Un DAG signifie :

  • Data Analysis Graph
  • Dynamic API Gateway
  • Distributed Access Grid
  • Directed Acyclic Graph

Question 3

Un DAG représente :

  • Une base SQL
  • Un pipeline de tâches
  • Un serveur web
  • Une API REST

Question 4

Une tâche (task) est :

  • Une unité de travail
  • Un serveur
  • Un cluster
  • Une base

Question 5

Airflow est écrit principalement en :

  • Java
  • Python
  • C++
  • Go

Question 6

Le scheduler sert à :

  • Supprimer les données
  • Compiler des programmes
  • Planifier les exécutions
  • Créer des dashboards

Question 7

Un opérateur (Operator) permet de :

  • Définir une tâche
  • Stocker des données
  • Créer une base
  • Installer Docker

Question 8

Airflow est souvent utilisé pour :

  • ETL / pipelines data
  • Jeux vidéo
  • Systèmes embarqués
  • OS

Question 9

Une dépendance entre tâches signifie :

  • Une erreur
  • Une suppression
  • Une API
  • L’ordre d’exécution

Question 10

L’interface Web Airflow permet :

  • Superviser les workflows
  • Compiler
  • Supprimer
  • Installer

Bravo ! Je vous invite à prendre le temps d’explorer les différentes possibilités de l'outil. Apache Airflow n'est pas qu'un simple outil, c'est une plateforme tout en un qui répond à vos caharges de travail de data engineering.
